# OSI 7계층, TCP UDP 정리

## 1. OSI 7계층

데이터 전송 프로토콜을 7단계로 정리해 놓은 것으로

7~5단계는 응용 계층으로 사용자가 데이터를 호출/전송 하거나 보안 관련 시스템을 말한다

4단계는 네트워크 계층으로 TCP를 활용해 네트워크를 연결하는 계층을 말한다

3단계는 IP 계층으로 세그먼트를 패킷으로 묶어 IP를 지정해 어디로 데이터를 보낼지 지정한다

2,1 단계는 MAC주소와 실제로 케이블을 활용해 데이터를 전송한다

## 2. TCP와 UDP의 차이점

가장 큰 차이점은 비연결형 프로토콜인지 연결형 프로토콜인지가 가장 큰 차이점 입니다.

TCP는 연결형 프로토콜로 3way handshake를 통해 클라이언트와 서버간의 연결을 확인 한 후 데이터를 송수신하지만 UDP는 비연결형 프로토콜로 연결을 확인하지 않고 데이터를 송수신 합니다

이로 인해 신뢰성과, 성능 차이가 발생하는데 TCP는 여기서 무결성, 송수신 입증, 데이터 순서 보장을 하지만 속도는 UDP보다 낮게 측정됩니다.

## 3. TCP와 3 way handshake

TCP의 라이프사이클은 연결 전송 연결해제로 구성되어 있는데 3 way는 연결 할때 사용하는 기술입니다.

구동 방식은 클라이언트에서 서버로 syn 패킷을 전송하고 서버는 응답으로 ack패킷과 서버의 syn를 클라이언트로 보냅니다.

그 후 클라이언트는 정상 응답일 시 다시 서버로 ack 패킷을 전송해 서버와 연결을 동작 시킵니다

이때 패킷에는 checksum, sequence number, flag, ack number가 들어가게 됩니다.
