# API란 무엇인가

## API(Application Programming Interface)

- API란  
  프로그램과 프로그램이 소통하기 위한 약속  
  어떤 기능을 어떻게 호출  
  어떤 데이터를 주고  
  어떤 형태로 응답 받을지를 명확하게 정의한 인터페이스

- 여기서 interface란?  
  내부 로직을 숨기고 사용자는 정의된 방식으로 사용할 수 있게 추상화 한 것을 말함

## API는 왜 나왔을까?

문제  
시스템이 커질수록 내부 구현에 의존하면 결합도가 높아지고 변경에 취약해지기 때문에, 이를 해결하기 위해 API가 등장했습니다.

1. 내부 구현을 직접 알아야 기능 사용 가능
2. 시스템이 커질수록 강한 결합도
3. 하나를 바꾸면 전부 깨질 수 있음

해결

1. 역할 분리  
   프론트 / 백엔드  
   서비스 / 서비스
2. 확장성  
   모바일 앱, 웹, 외부 서비스에서 동일 기능 사용
3. 보안  
   내부로직 숨기고 창구만 제공
4. 협업  
   api 명세만 맞으면 병렬 개발 가능

## 그럼 REST API는 뭘까?

API를 설계하는 아키텍처 스타일 이다.

REST란?
자원을 URI로 표현하고, 행위는 Method로 표현

여기서 자원이란?  
시스템에서 관리하고 싶은 대상 (데이터, 개념, 상태 등)  
아래 예시에서는 user가 자원
그래서 이 자원을 아래와 같이 표현하는게 REST

```http
GET /users/1
POST /users
DELETE /users/1
```

RESTful 하다는 것은?
자원을 URI로 표현하고, HTTP Method와 상태 코드를 의미에 맞게 사용하며,일관된 규칙을 지키는 것이다

## API와 라이브러리의 차이는?

API는 “무엇을 할 수 있는지”를 정의하고, 라이브러리는 “그걸 어떻게 구현했는지”를 제공한다.

API = 규약  
라이브러리 = 구현체
