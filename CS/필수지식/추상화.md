# 추상화란?

복잡한 내부 구현을 숨기고 사용자에게 필요한 핵심 개념과 사용 방법만 노출하는 것

## 추상화와 캡슐화의 차이점

### 추상화

1. 무엇을 할 수 있는가를 정의
2. 복잡한 시스템에서 핵심 개념과 행위만 뽑아내는 것
3. 사용자가 알아야 할 것만 노출
4. 설계 단계의 개념

### 캡슐화

1. 그걸 어떻게 안전하게 숨길 것인가를 정의
2. 데이터와 로직을 하나로 묶고, 외부 접근을 제한하는것
3. 내부 상태를 보호
4. 어떻게 구현했는가를 숨김
5. 구현/안정성 관점

## 코드 예시

### 추상화

```js
fetch("/users/1");
```

### 캡슐화

#age 직접접근 x => 메서드를 통해서만 접근

```js
class User {
  #age;

  getAge() {
    return this.#age;
  }
}
```

## 추상화의 활용 예시

HTTP Method  
http method는 자원에 대해 수행할 수 있는 행위를 추상화한 개념입니다.  
사용자는 GET, POST, DELETE, PATCH와 같은 정의된 Method만 알면 되고,  
실제로 서버 내부의 어떤 로직이나 DB작업이 수행되는지 알 필요 없이 사용할 수 있기 때문에 추상화에 해당합니다.
