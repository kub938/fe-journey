# 동시성 & 멀티쓰레딩

## 동시성 프로그래밍 (concurrency)

동시에 프로그램을 실행함

### 구현법

대표적으로 스레딩 모델

1. 멀티 스레딩

   - a,b,c,d 병렬로 처리
   - 논리적인 동시성(concurrency)을 제공
   - 싱글 코어에서는 운영체제가 빠르게 번갈아 실행하는 시분할방식(time slicing)을 사용
     - 이때 처리 방식은 a조금 -> b조금 -> c조금 -> d조금 -> ... 이런식 짧은 주기로 짧게씩 실행되서 병렬로 처리 되는것 처럼 보임
   - 멀티코어에서는 병렬로 가능 하긴 함

   - 단점
   - 여러 스레드가 공유자원을 동시 접근
     -> race condition, deadlock 등과 같은 오류 발생 </br>
     Race Condition: 여러 스레드가 동시에 같은 데이터를 읽고/수정할 때, 실행 순서에 따라 결과가 달라지는 문제 </br>
     Deadlock: 서로 다른 스레드가 서로가 가진 자원의 락을 기다리며 무한 대기 상태에 빠짐
     스레드 A는 자원1 잠금 -> 자원2 기다림/
     스레드 B는 자원2 잠금 -> 자원1 기다림
   - 디버깅, 티스트 어려움
   - 메모리사용량 증가
